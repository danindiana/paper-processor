## Findings / Observations

1. **Single-Model Residency Issue (RTX 3080):** During integration testing, it was observed that only one model stayed loaded on the RTX 3080 despite the script's optimized mapping.
2. **Root Cause - KV Cache Overhead:** The system's `ollama.service` had `OLLAMA_NUM_PARALLEL=2` configured. This doubled the KV cache size, causing `deepseek-r1:8b` to reserve **9.6 GB** of the available 10 GB. This left insufficient room for `qwen3.5:4b` (3.4 GB), resulting in immediate model eviction.
3. **Resolution - Service Reconfiguration:** To maintain true multi-model residency on a 10GB card, internal parallelism must be reduced. The following configuration is required for the 3080's service:
   - `OLLAMA_NUM_PARALLEL=1` (Reduces KV cache footprint)
   - `OLLAMA_MAX_LOADED_MODELS=2` (Allows multiple models to co-exist)
   The secondary GPU (with 16GB) can maintain `OLLAMA_NUM_PARALLEL=2`.

4. **Final Applied Systemd Overrides:**

   **GPU 0 (RTX 3080, Port 11434):**
   ```ini
   [Service]
   Environment="OLLAMA_HOST=127.0.0.1:11434"
   Environment="CUDA_VISIBLE_DEVICES=0"
   Environment="OLLAMA_NUM_PARALLEL=1"
   Environment="OLLAMA_MAX_LOADED_MODELS=2"
   ```

   **GPU 1 (RTX 5080, Port 11435):**
   ```ini
   [Service]
   Environment="OLLAMA_HOST=0.0.0.0:11435"
   Environment="CUDA_VISIBLE_DEVICES=1"
   Environment="OLLAMA_NUM_PARALLEL=2"
   Environment="OLLAMA_MAX_LOADED_MODELS=2"
   ```
